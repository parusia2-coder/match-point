# 🏸 Minton-Tennis (배드민턴/테니스 통합) 유지보수 및 관리 지침서

본 문서는 향후 원활하고 효율적인 시스템 유지보수를 위해 작성되었습니다. 시간이 지나 누군가 시스템을 수정하거나 확장할 때 길잡이가 되는 핵심 기술 문서입니다.

---

## 1. 🏗️ 전체 시스템 구조 (Tech Stack)

이 시스템은 가장 빠르고 비용이 저렴하며 전 세계 어디서든 접속이 쾌적한 **클라우드플레어 (Cloudflare)** 생태계 기반으로 구축되었습니다.

- **서버 및 호스팅**: Cloudflare Pages (프론트엔드 + 백엔드 API 통합 호스팅)
- **백엔드 프레임워크**: Hono (빠르고 가벼운 타입스크립트 기반 웹 프레임워크)
- **데이터베이스**: Cloudflare D1 (SQLite 기반의 분산형 관계형 DB)
- **프론트엔드**: Vanilla JS + 순수 CSS (외부 프레임워크 의존성을 줄여 초경량화)
- **빌드 도구**: Vite (초고속 번들러)

---

## 2. 📂 핵심 폴더 및 구조 가이드

코드를 고칠 때는 아래의 역할을 참고하여 해당 폴더의 파일만 열어보시면 됩니다.

1. **`⚙️ /src/` (백엔드 서버 & API)**
   - 앱의 "두뇌" 역할을 합니다. 데이터베이스에서 정보를 가져오거나 저장하는 로직이 있습니다.
   - `/src/index.tsx`: 메인 라우터 (주소창 URL 정리 및 HTML 화면 연결)
   - `/src/routes/`: 참가자, 경기대진표, 종목 등 기능별로 쪼개놓은 API 통신 규약
2. **`🎨 /public/static/` (프론트엔드 화면 디자인)**
   - 앱의 "얼굴과 버튼" 역할을 합니다.
   - `/public/static/style.css`: 글씨 크기, 버튼 색상, 모바일 반응형 디자인
   - `/public/static/app.js`: 관리자 대시보드 화면 및 리스트, 모달창 렌더링 로직
   - `/public/static/court.js`: 코트 점수판 전용 로직 (점수 올리기, 승리 판정)
3. **`🗃️ /migrations/` (데이터베이스 구조 설계도)**
   - 테이블(표) 설계도. 새로운 입력 항목(예: "참가자 혈액형")이 필요해지면 이곳에 SQL 스크립트를 추가하여 업데이트합니다.

## 3. 🛠️ 가장 자주 발생할 수정 요청 가이드

시간이 지나면 이런 요구사항들이 주로 발생합니다. 다음 절차를 따르세요.

### Q1. 화면 글씨 크기를 키우거나, 버튼 색깔을 바꾸고 싶어요.
- **수정 위치**: `public/static/style.css`
- **방법**: `:root` 상단의 `--primary` 컬러값이나, 반응형 구문을 다루는 `@media (max-width: 768px)` 아래의 블록을 수정하세요.

### Q2. "배드민턴 코트 점수판"의 승리 판정 기준을 25점에서 21점으로 바꾸고 싶어요.
- **수정 위치**: `public/static/court.js`
- **방법**: `checkBadmintonGameWinner()` 함수 내부에 기재된 `limit = 25` 변수를 찾아 원하는 점수로 수정하세요. (테니스는 `checkTennisGameWinner`)

### Q3. "참가자" 목록에 "소속 지역"이라는 정보를 새로 추가하고 싶어요.
- **수정 위치 (DB)**: `migrations` 폴더에 `0002_add_region.sql` 등 새 파일을 만들고 `ALTER TABLE participants ADD COLUMN region TEXT;` 작성 후, D1 터미널 `migrations apply` 명령 실행.
- **수정 위치 (화면)**: `public/static/app.js` 내의 `renderParticipants()` 함수에 테이블 텍스트 `<th>소속 지역</th>` 및 `${p.region}` 코드 추가.

### Q4. 경기 결과 "승자 표시 대진표 자동 이동" 기능 규칙을 변경하고 싶어요.
- **수정 위치**: `/src/routes/matches.ts`
- **방법**: `autoAdvanceBracket()` 함수 내부의 승자 이동 로직을 변경하시면 됩니다.

### Q5. QR 코드 모달의 URL이 달라요.
- **수정 위치**: `public/static/court.js` → `showQRCodeModal()` 함수
- **3가지 모드**:
  - 🏸 **심판용**: `/court?tid={X}&court={Y}` — 터치로 점수 입력 가능
  - 📺 **관람용**: `/court?tid={X}&court={Y}&locked=1` — 동일 점수판, 터치 완전 비활성 (대형 모니터용)
  - ⌚ **워치용**: `/watch?tid={X}&court={Y}` — 스마트워치/소형화면 최적화 페이지

### Q6. 코트 점수판에서 상위 화면으로 이동하는 버튼 위치
- 페이지 네비게이션 구조:
  ```
  /?tid=1 (메인 대회 관리)
    → /court?tid=1 (코트 센터)    ← 대회 관리 버튼 [좌상단]
      → /court?tid=1&court=1      ← 코트 센터 버튼 [상단 바]
         (코트 점수판 · 대기/경기)
  ```
- **수정 위치**: `public/static/court.js`
  - 코트 센터 뒤로가기: `renderCourtCenter()` 함수 내 `← 대회 관리` 버튼
  - 점수판 뒤로가기: `renderScoreboard()` 함수 내 `← 코트 센터` 버튼
  - 함수: `window.goToCourtCenter()`

## 4. 🖨️ 개발/테스트 및 배포 (명령어 치트시트)

### ⚡ 원클릭 자동 배포 (권장)

코드 수정 후 아래 명령어 하나로 빌드 + static 파일 동기화 + Cloudflare 배포가 한 번에 됩니다.

```powershell
# 기본 실행
.\deploy.ps1

# 변경 내용 메모 포함
.\deploy.ps1 -Message "코트 점수판 버그 수정"
```

> ⚠️ **중요**: `public/static/` 파일(court.js, style.css 등)을 수정했을 때는
> 반드시 `deploy.ps1`을 사용하세요. `npm run build`만 하면 static 파일이
> dist에 반영되지 않아 서버에 변경사항이 올라가지 않습니다.

### 수동 3단계 배포

1. **로컬 테스트**: `npm run dev` → `http://localhost:8787`
2. **빌드**: `npm run build` (dist/_worker.js 생성)
3. **Static 동기화**: `Copy-Item -Path public\static\* -Destination dist\static\ -Force`
4. **배포**: `npm run deploy`

> **💡 팁:** Cloudflare Dashboard → Workers & Pages → D1 에서 DB를 직접 조회/수정할 수 있습니다.
