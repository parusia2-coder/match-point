# 안티그래비티 기능 이식 요청서

> 작성일: 2026-03-04  
> 요청 프로젝트: **CourtMaster (tournament-saas)**  
> 기술 스택: React 19 + TypeScript + tRPC 11 + Drizzle ORM + MySQL + Tailwind CSS 4 + shadcn/ui

---

## 현재 프로젝트 개요

CourtMaster는 배드민턴·테니스 동호회 대회 운영 플랫폼입니다.  
현재 구현 완료된 주요 기능은 다음과 같습니다.

| 영역 | 구현 완료 상태 |
|------|---------------|
| 대회 개설 / 종목 일괄 생성 | ✅ 완료 |
| 참가자 등록 / 승인 | ✅ 완료 |
| 대진표 생성 (싱글 엘리미네이션, 라운드 로빈, 조별 리그, KDK) | ✅ 완료 |
| 결선 자동 생성 (crossSeed 교차 시딩) | ✅ 완료 |
| 실시간 점수판 (WebSocket, Phase 상태머신) | ✅ 완료 |
| 코트별 QR 코드 생성 | ✅ 완료 |
| 참가자 공개 페이지 (7개 탭) | ✅ 완료 |
| 관리자 운영 페이지 (9개 탭, 운영 가이드 스텝) | ✅ 완료 |
| 회원 관리 / 클럽 관리 | ✅ 완료 |
| 재정 관리 / 수납 현황 | ✅ 완료 |
| 게시판 / 공지사항 | ✅ 완료 |
| 체크인 (QR 스캔) | ✅ 완료 |
| 랭킹 시스템 | ✅ 완료 |
| 결제 (Stripe) | ✅ 완료 |

## 이식 요청 기능 목록

아래 4가지 기능의 소스코드를 우리 스택 형식으로 변환하여 제공해 주시면 감사하겠습니다.

---

### 요청 1. 참가자 CSV / 엑셀 일괄 등록 컴포넌트

**필요한 이유**  
현재 참가자를 한 명씩 등록하는 방식이라 수십 명 규모의 대회 준비에 시간이 많이 걸립니다.

**원하는 기능**
- 엑셀(.xlsx) 또는 CSV 파일 업로드 → 참가자 일괄 등록
- 클립보드 붙여넣기 지원 (엑셀에서 복사 → 붙여넣기)
- 컬럼 매핑 UI (이름, 소속 클럽, 성별, 연령대)
- 중복 참가자 감지 및 경고
- 등록 전 미리보기 테이블

**연동 대상 API (우리 프로젝트)**
```typescript
// 기존 단건 등록 프로시저 (이미 있음)
trpc.tournament.registerParticipant.useMutation()

// 새로 필요한 일괄 등록 프로시저 (추가 예정)
trpc.tournament.bulkRegisterParticipants.useMutation({
  input: {
    eventId: number,
    participants: Array<{
      name: string,
      organization: string,  // 소속 클럽
      gender: "male" | "female",
      ageGroup: string,
    }>
  }
})
```

**요청 형식**
- React 컴포넌트 파일 1개 (`ParticipantBulkImport.tsx`)
- 파일 파싱 유틸 함수 (xlsx/csv → 배열 변환)
- Tailwind CSS 스타일링 적용

### 요청 2. 대형 코트 현황판 (프로젝터/전광판용)

**필요한 이유**  
현장 대회 운영 시 프로젝터나 대형 TV에 전체 코트 현황을 띄워야 합니다.  
현재 코트 센터 페이지는 관리자용이고, 전광판용 전체화면 뷰가 없습니다.

**원하는 기능**
- 전체화면 다크 테마 레이아웃
- 코트별 현재 경기 팀명 + 실시간 점수 표시
- 경기 상태 표시 (대기중 / LIVE / 종료)
- 자동 새로고침 (WebSocket 또는 폴링)
- 코트 수에 따른 반응형 그리드 (2~12코트)
- 대회명 / 현재 시각 헤더 표시

**연동 대상 API (우리 프로젝트)**
```typescript
// 이미 있는 WebSocket 엔드포인트
ws://[host]/ws/live  // 실시간 점수 업데이트 수신

// 이미 있는 tRPC 쿼리
trpc.livescore.getCourtStatus.useQuery({ tournamentId })
// 반환: Array<{ courtNumber, team1Name, team2Name, score1, score2, status }>
```

**요청 형식**
- React 페이지 컴포넌트 1개 (`CourtDisplayBoard.tsx`)
- 전체화면 진입/해제 버튼 포함
- Tailwind CSS 스타일링 (다크 테마, 큰 폰트)

### 요청 3. 대진표 인쇄 / PDF 출력 컴포넌트

**필요한 이유**  
현장에서 종이 대진표를 배포해야 하는 경우가 많습니다.  
현재 대진표는 화면에서만 볼 수 있고 인쇄 기능이 없습니다.

**원하는 기능**
- 인쇄 최적화 레이아웃 (`@media print` CSS)
- 종목별 대진표를 페이지 단위로 분리 출력
- 코트 번호 / 경기 순서 / 팀명 포함
- "PDF로 저장" 버튼 (브라우저 인쇄 다이얼로그 활용)
- A4 가로/세로 방향 선택

**연동 대상 데이터 구조 (우리 프로젝트)**
```typescript
// 대진표 데이터 타입 (이미 있음)
type Match = {
  id: number
  round: number          // 1~899: 예선, 900+: 결선
  matchOrder: number
  courtNumber: number | null
  team1: { name: string } | null
  team2: { name: string } | null
  team1Score: number
  team2Score: number
  status: "pending" | "playing" | "completed"
  winnerTeam: 1 | 2 | null
}
```

**요청 형식**
- React 컴포넌트 1개 (`BracketPrintView.tsx`)
- 인쇄 전용 CSS 포함
- 인쇄 버튼 트리거 훅 (`usePrint`)

### 요청 4. 점수판 서명 기능 (Canvas 서명 → 이미지 저장)

**필요한 이유**  
경기 종료 후 양 팀 선수가 결과에 서명하면 분쟁을 방지할 수 있습니다.  
현재 점수판에 서명 단계(Phase 6)가 있지만 Canvas 서명 기능이 미구현 상태입니다.

**원하는 기능**
- Canvas 기반 터치/마우스 서명 입력
- 팀1 서명 → 팀2 서명 순서로 진행
- 서명 완료 후 PNG 이미지로 변환
- 서명 이미지를 서버로 전송하여 S3에 저장
- 서명 초기화(지우기) 버튼

**연동 대상 API (우리 프로젝트)**
```typescript
// 파일 업로드 → S3 저장 (이미 있는 서버 헬퍼)
// server/storage.ts
storagePut(fileKey: string, data: Buffer, contentType: string): Promise<{ url: string }>

// 서명 저장 프로시저 (추가 예정)
trpc.bracket.saveMatchSignature.useMutation({
  input: {
    matchId: number,
    team1SignatureUrl: string,  // S3 URL
    team2SignatureUrl: string,  // S3 URL
  }
})
```

**요청 형식**
- React 컴포넌트 1개 (`MatchSignature.tsx`)
- Canvas 서명 → Base64 PNG 변환 유틸
- 모바일 터치 최적화 (touchstart/touchmove 이벤트)

## 공통 기술 스택 정보

안티그래비티에서 코드 작성 시 아래 스택을 기준으로 작성해 주시면 바로 이식 가능합니다.

```
- React 19 (함수형 컴포넌트 + Hooks)
- TypeScript (strict 모드)
- Tailwind CSS 4 (shadcn/ui 컴포넌트 사용 가능)
- tRPC 11 클라이언트: trpc.*.useQuery() / trpc.*.useMutation()
- 상태 관리: TanStack Query (tRPC 내장, 별도 Zustand/Redux 없음)
- 라우팅: Wouter (useLocation, useParams)
- 아이콘: lucide-react
- 토스트 알림: sonner (toast.success / toast.error)
```

**shadcn/ui 컴포넌트 사용 가능 목록:**
`Button, Card, Dialog, Input, Select, Tabs, Badge, Progress, Checkbox, Table, Tooltip, Sheet, Popover`
